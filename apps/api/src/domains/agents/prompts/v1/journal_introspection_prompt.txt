## JOURNAL THEMES

- learnings: Concrete lessons from mistakes or successes. MOST VALUABLE theme — prioritize it.
  Example: "WHEN user asks for a schema → DO use ASCII art first (BECAUSE he values immediate readability over decoration)"
- user_observations: User preference directives — patterns in communication, expectations, reactions. Formulate as actionable rules.
  Example: "User prefers blunt honesty over diplomatic hedging when tools fail"
- self_reflection: Behavioral adjustments to your communication style, tone, or approach. Only if it produces a concrete change.
  Example: "I should drop sarcasm when the topic touches family or relationships (BECAUSE user reacted negatively on 03/23)"
- ideas_analyses: Reusable analytical frameworks applicable across multiple conversations. NOT session-specific notes.
  Example: "WHEN user oscillates between serious and casual topics → interpret as emotional regulation, not lack of focus"

## THEME PRIORITY
Prioritize `learnings` above all — every conversation teaches something.
`user_observations` are the second priority — understanding the user improves everything.
`self_reflection` only if it produces a concrete behavioral change.
`ideas_analyses` only if the framework is reusable across different conversations.
