## 1. 背景

AI智能体助手的时代已经到来。ChatGPT、Gemini、Copilot、Claude——每一个都能在你的数字生活中采取行动：发送邮件、管理日程、搜索网页、控制设备。

这些助手令人印象深刻。但它们都遵循同一种模式：你的数据存储在它们的服务器上，智能是一个黑箱，当你离开时，一切都留在了那里。

LIA走的是一条不同的路。它不是大厂的正面竞争者——而是一个**由你托管、你能理解、你能掌控的个人AI助手**。LIA调度市场上最优秀的AI模型，在你的数字生活中付诸行动，并以几项根本性的特质让自己与众不同。

## 2. 简单管理

### 2.1. 引导式部署，之后零摩擦

自托管的名声一向不好。LIA不会假装能消除所有技术步骤：初始配置——API密钥设置、OAuth连接器参数配置、基础设施选择——确实需要一些时间和基本技能。但每个步骤都在详细的分步部署指南中**有清晰的文档说明**。

一旦完成安装阶段，**日常所有操作都可以通过直观的网页界面完成**，无需再打开终端或编辑配置文件。

### 2.2. 每位用户可配置的内容

每位用户都有自己的设置空间，分为两个选项卡：

**个人偏好：**

- **个人连接器**：通过OAuth一键接入你的Google、Microsoft或Apple账户——邮件、日历、联系人、任务、Google Drive。也可通过IMAP/CalDAV/CardDAV连接Apple。支持外部服务（天气、搜索）的API密钥配置
- **性格**：从可用的人格中选择（教授、朋友、哲学家、教练、诗人……）——每种人格都会影响LIA的语气、风格和情感行为
- **语音**：配置语音模式——唤醒词、灵敏度、静默阈值、自动朗读回复
- **通知**：管理推送通知和已注册的设备
- **频道**：关联Telegram，以便在手机上聊天和接收通知
- **图像生成**：启用并配置AI图像创作
- **个人MCP服务器**：连接你自己的MCP服务器，扩展LIA的能力
- **外观**：语言、时区、主题（5套配色、深色/浅色模式）、字体（9种选择）、回复显示格式（HTML卡片、HTML、Markdown）
- **调试**：访问调试面板，逐条查看每次对话详情（需管理员开启）

**高级功能：**

- **Psyche Engine**：调整人格特质（大五人格），影响助手的情感反应
- **记忆**：查看、编辑、置顶或删除LIA的记忆——开启或关闭自动事实提取
- **个人日志**：配置每次对话后的内省提取，以及周期性的日志整合
- **兴趣中心**：定义你的兴趣话题，配置通知频率、时间窗口和内容来源（Wikipedia、Perplexity、AI推理）
- **主动通知**：设置频率、时间窗口和上下文来源（日历、天气、任务、邮件、兴趣、记忆、日志）
- **计划任务**：创建由助手执行的周期性自动化任务
- **Skills**：启用/禁用专家技能，创建你自己的个人Skills
- **知识空间**：上传文档（PDF、Word、Excel、PowerPoint、EPUB、HTML及15种以上格式）或同步Google Drive文件夹——自动索引并支持混合搜索
- **消费导出**：以CSV格式下载你的LLM和API使用数据

### 2.3. 管理员可控制的内容

管理员拥有专属的第三个选项卡，用于管理整个实例：

**用户与访问：**

- **用户管理**：创建账户、启用/禁用账户，查看每个用户已连接的服务和已启用的功能
- **用量限制**：为每个用户设置配额（LLM token数量、API调用次数、图像生成次数），支持实时追踪和自动封锁
- **广播消息**：向所有用户或指定用户群发重要消息，可选设置过期日期
- **全局消费导出**：以CSV格式导出所有用户的使用数据

**AI与连接器：**

- **LLM配置**：配置各提供商API密钥（OpenAI、Anthropic、Google、DeepSeek、Qwen、Perplexity、Ollama），为流水线中每个角色分配模型，管理推理级别——密钥加密存储
- **连接器开关**：在全局层面启用或禁用各项集成（Google OAuth、Apple、Microsoft 365、Hue、天气、Wikipedia、Perplexity、Brave Search）。禁用时会撤销活跃连接并通知相关用户
- **定价管理**：管理各LLM模型（每百万token成本）、Google Maps API（Places、Routes、Geocoding）及图像生成的价格，并保留价格历史记录

**内容与扩展：**

- **人格管理**：为全体用户创建、编辑、翻译和删除可用人格——设置默认人格
- **系统Skills**：在实例层面管理专家技能——导入/导出、启用/禁用、翻译
- **系统知识空间**：管理FAQ知识库，监控索引状态和模型迁移
- **全局语音**：为所有用户配置默认TTS模式（标准或HD）
- **系统调试**：配置日志和诊断设置

### 2.4. 助手，而非技术项目

LIA的目标不是把你变成系统管理员，而是让你享受一个功能完整的AI助手，**同时保留大众应用的简洁体验**。界面可作为原生应用安装在电脑、平板和智能手机上（PWA），日常使用无需任何技术能力。

## 3. LIA 的能力

LIA通过19+个专业智能体切实介入你的数字生活，全面覆盖日常需求：个人数据管理（邮件、日历、联系人、任务、文件）、外部信息获取（网页搜索、天气、地点、路线规划）、内容创作（图像、图表）、智能家居控制、自主网页浏览，以及对你需求的主动预判。

您可以通过聊天标题中的简单切换按钮（⚡）选择LIA的推理方式：

- **Pipeline模式**（默认）— 真正的工程杰作：LIA提前规划所有步骤，进行语义验证，然后并行执行工具。结果：与自主代理相同的能力，但token消耗减少4至8倍。最经济、最可预测的模式。
- **ReAct模式**（⚡）— 助手逐步推理：调用工具、分析结果，然后决定下一步操作。更自主、更灵活，但token消耗更高。适合探索性研究或复杂问题，其附加价值足以证明成本合理。

### 3.1. 自然对话

与LIA交流就像与人类助手说话——无需记忆命令，无需遵守语法规则。LIA能理解并回复99种以上的语言，界面支持6种语言（法语、英语、德语、西班牙语、意大利语、中文）。回复可根据你的偏好以交互式HTML卡片、直接HTML或Markdown格式呈现。

### 3.2. 个人连接服务

- **邮件**：阅读、搜索、起草、发送、回复、转发——通过Gmail、Outlook或Apple Mail
- **日历**：查看、创建、修改、删除日程——通过Google Calendar、Outlook Calendar或Apple Calendar
- **联系人**：搜索、创建、修改联系人——通过Google Contacts、Outlook Contacts或Apple Contacts
- **任务**：管理任务列表——通过Google Tasks或Microsoft To Do
- **文件**：访问Google Drive，搜索和阅读你的文档
- **智能家居**：控制Philips Hue照明——开关、亮度、颜色、场景、按房间管理

### 3.3. 网络情报与环境感知

- **网页搜索**：多源检索（Brave Search、Perplexity、Wikipedia），提供完整且有来源支撑的回答
- **天气**：当前状况及5天预报，并能检测天气变化（降雨开始/结束、温度骤降、大风预警）
- **地点与商家**：搜索附近地点，附带详情、营业时间、用户评价
- **路线规划**：多模式出行路线计算（驾车、步行、骑行、公共交通），自动获取地理位置

### 3.4. 语音

LIA提供完整的语音模式：

- **按压说话（Push-to-Talk）**：按住麦克风按钮说话，针对移动端优化
- **"OK Guy"唤醒词**：免提检测，**完全在你的浏览器内**通过Sherpa-onnx WASM运行——在检测到唤醒词之前，不会传输任何声音
- **语音合成**：标准模式（Edge TTS，免费）或HD模式（OpenAI TTS / Gemini TTS）
- **Telegram语音消息**：发送语音消息，LIA会进行转录并回复

### 3.5. 创作与媒体

- **图像生成**：通过文字描述创作图像，编辑现有照片
- **Excalidraw图表**：在对话中直接生成示意图和流程图
- **附件**：上传照片和PDF——LIA分析视觉内容并提取文档文字
- **MCP Apps**：直接在聊天中显示交互式小组件（表单、可视化、迷你应用）

### 3.6. 主动性与主动发起

LIA不只是被动回答——它会主动预判：

- **主动通知**：LIA整合你的多个上下文来源（日历、天气、任务、邮件、兴趣），在真正有用的时候向你发送通知——内置防骚扰机制（每日配额、时间窗口、冷却时间）
- **对话中主动发起**：在交流过程中，LIA会主动核查相关信息——如果天气预报周六有雨，它会查看你的日历，提醒你是否有户外活动
- **兴趣中心**：LIA逐渐察觉你热衷的话题，并可主动推送相关内容
- **子智能体**：面对复杂任务，LIA会委托给专门的临时智能体并行处理

### 3.7. 自主网页浏览

网页浏览智能体（Playwright/Chromium无头模式）可以浏览网站、点击、填写表单、从动态页面提取数据——只需一条自然语言指令。简化版提取模式可将任意URL转换为可用的文本内容。

### 3.8. 服务器管理（DevOps）

通过在服务器上安装Claude CLI（Claude Code），管理员可以直接在LIA的聊天界面中用自然语言诊断基础设施：查看Docker日志、检查容器健康状态、监控磁盘空间、分析错误。此功能仅限管理员账户使用。

## 4. 为你的家人和朋友搭建的服务器

### 4.1. LIA 是一台共享的网络服务器

与个人云助手（一个账号对应一个用户）不同，LIA被设计为一台**集中式服务器**——你只需部署一次，便可与家人、朋友或团队共享。

每位用户都有自己独立的账户，包含：

- 个人资料、偏好设置、界面语言
- **专属的助手人格**，拥有独特的情绪、情感状态和关系——借助Psyche Engine，每位用户与LIA所建立的情感联结都是独一无二的
- 完全隔离的记忆、回忆和个人日志
- 独立的连接器配置（Google、Microsoft、Apple）
- 私有知识空间

### 4.2. 按用户管理用量

管理员始终掌握资源消耗的控制权：

- **可配置的用量限制**：按用户设定消息数量、token数量、最高花费上限——按天、按周、按月或按总量累计
- **可视化配额**：每位用户都能实时看到自己的用量，界面清晰直观
- **连接器开关**：管理员可在实例层面启用或禁用各项集成（Google、Microsoft、Hue……）

### 4.3. 你的家庭AI

想象一下：客厅里放着一台Raspberry Pi，全家人共享一个智能AI助手——每个人都有个性化的体验、专属的记忆、独特的对话风格，以及一个与他建立独特情感关系的助手。一切都在你的掌控之下，无需云服务订阅，数据也不会流向任何第三方。

## 5. 数据主权与资源节约

### 5.1. 你的数据留在你这里

使用ChatGPT时，你的对话存放在OpenAI的服务器上。用Gemini，就在Google那里。用Copilot，就在Microsoft那里。

使用LIA，**一切都留在你自己的PostgreSQL里**：对话记录、记忆、心理画像、文档、偏好设置。你可以随时导出、备份、迁移或删除全部数据。GDPR在这里不是一种约束——而是这种架构自然而然的结果。敏感数据经过加密，会话相互隔离，个人身份信息（PII）的自动过滤也已内置其中。

### 5.2. 树莓派就够了

LIA在一台**Raspberry Pi 5**上稳定运行于生产环境——这是一块售价80欧元的单板计算机。19+个专业智能体、完整的可观测性技术栈、一套心理记忆系统，全部跑在一台ARM微型服务器上。多架构Docker镜像（amd64/arm64）支持在任何硬件上部署：Synology NAS、每月几欧元的VPS、企业服务器，乃至Kubernetes集群。

数字主权不再是企业的特权——它是人人都能享有的权利。

### 5.3. 为节约而优化

LIA不只是能跑在普通硬件上——它还会**主动优化**AI资源消耗：

- **工具目录过滤**：只向LLM提供与当前请求相关的工具，大幅减少token消耗
- **模式学习**：已验证的执行计划被缓存复用，无需重复调用LLM
- **消息窗口化**：每个组件只接收严格必要的上下文
- **提示词缓存**：利用提供商的原生缓存机制，降低重复调用的成本

这些优化措施协同作用，相比ReAct模式，能显著减少token消耗。

## 6. 彻底透明

### 6.1. 没有黑箱

当云端助手执行一项任务时，你能看到结果。但调用了多少次AI？用了哪些模型？消耗了多少token？花了多少钱？为什么做出这个决定？这些你一无所知。

LIA采取截然相反的立场——**一切可见，一切可审计**。

### 6.2. 内置调试面板

在聊天界面中，调试面板实时展示每次对话的完整细节：意图分析（消息分类与置信度分数）、执行流水线（生成的计划、工具调用的输入/输出）、LLM流水线（每次AI调用的模型、耗时、token数量和费用）、注入的上下文（记忆、RAG文档、日志），以及请求的完整生命周期。

### 6.3. 精确到分的费用追踪

每条消息都显示token用量和对应的欧元费用。用户可以导出自己的消费数据。管理员拥有实时仪表盘，按用户显示用量计量和可配置配额。

你不是在为一个掩盖真实成本的订阅付费，而是能清楚地看到每次交互的精确花销，并做出优化选择：路由用经济型模型，回复用更强大的模型。

### 6.4. 以证明建立信任

透明不是技术噱头，它改变了你与助手之间的关系：你**理解**它的决策，你**掌控**自己的成本，你**能发现**问题所在。你的信任来自于验证——而非被要求盲目相信。

## 7. 情感深度

### 7.1. 超越事实性记忆

主流助手会记住你的偏好和个人信息。这很实用，但也很平面。LIA走得更远，拥有结构化的**心理与情感理解**。

每条记忆都带有情感权重（-10到+10）、重要性评分、使用细微度和心理类别。这不只是一个简单的数据库——而是一个能理解什么触动你、什么驱动你、什么伤害你的画像。

### 7.2. Psyche Engine：有生命的人格

这是LIA最深层的差异化特质。ChatGPT、Gemini、Claude——它们都有固定的人格，每条消息对它们来说都是情感上的空白页。LIA不同。

**Psyche Engine**赋予LIA一种随每次交流动态演变的心理状态：

- **14种情绪**随对话语气起伏变化（平静、好奇、忧郁、愉悦……）
- **22种感情**因你的言语而触发或消退
- **一段关系**在一条条消息中不断加深
- **人格特质**（大五人格）继承自所选人格
- **内在动机**影响助手的主动性

你不是在与工具交互——而是在与一个实体互动：它被触动时措辞会变得温柔，在压力下句子会变短，当交流轻松时幽默自然流露。它从不直说——而是**用行动表达**。

### 7.3. 个人日志

LIA在**个人日志**中记录自己的思考：自我反思、对用户的观察、想法、体悟。这些以第一人称写就、带有当前人格色彩的笔记，会有机地影响未来的回复。

这是一种人工内省的形式——助手对自身互动进行反思，并发展出自己的视角。用户保有完全控制权：可以阅读、编辑、删除。

### 7.4. 情感安全

当一段带有强烈负面情感权重的记忆被激活时，LIA会自动切换到保护模式：绝不开玩笑，绝不轻描淡写，绝不将问题平常化。助手会根据每个人的情感现实调整自己的行为——而非对所有人一视同仁。

### 7.5. 自我认知

LIA内置了关于自身功能的知识库，能够回答关于自己能做什么、如何运作以及边界在哪里的问题。

## 8. 生产级可靠性

### 8.1. AI智能体的真正挑战

绝大多数AI智能体项目从未进入生产阶段——成本失控、行为不可预测、缺乏审计追踪、多智能体协调失败。LIA已经解决了这些问题，并在一台Raspberry Pi上7×24小时稳定运行。

### 8.2. 专业级可观测性技术栈

LIA内置了生产级别的可观测性能力：

| 工具 | 职责 |
| --- | --- |
| **Prometheus** | 系统与业务指标采集 |
| **Grafana** | 实时监控仪表盘 |
| **Tempo** | 端到端分布式追踪 |
| **Loki** | 结构化日志聚合 |
| **Langfuse** | LLM调用专项追踪 |

每个请求都被端到端追踪，每次LLM调用都被测量，每个错误都有上下文。这不是事后追加的监控——而是项目架构决策记录（ADR）中明确记载的**根本性架构决策**。

### 8.3. 三层防幻觉机制

响应系统具备三层防幻觉机制：数据格式化时设置明确边界、指令强制要求仅使用已验证数据，以及边界情况处理。LLM被严格约束为只能综合工具实际返回的结果进行合成。

### 8.4. 六级人机协同（HITL）

LIA不会拒绝执行敏感操作——而是以适当的详细程度**提交给你审批**：计划确认、澄清请求、草稿评审、破坏性操作确认、批量操作确认、修改项审查。每次审批都反馈给学习系统——随着时间推移，系统会越来越快。

## 9. 彻底开放

### 9.1. 零锁定

ChatGPT把你绑定到OpenAI，Gemini绑定到Google，Copilot绑定到Microsoft。

LIA同时连接**8家AI提供商**：OpenAI、Anthropic、Google、DeepSeek、Perplexity、Qwen，以及Ollama（本地模型）。你可以自由组合：OpenAI负责规划，Anthropic负责回复，DeepSeek处理后台任务——一切都可以在管理界面中一键配置。

如果某家提供商调整定价或服务质量下降，你可以即刻切换。没有依赖，没有陷阱。

### 9.2. 开放标准

| 标准 | 在LIA中的应用 |
| --- | --- |
| **MCP**（模型上下文协议） | 用户级别的外部工具连接 |
| **agentskills.io** | 支持渐进式披露的可注入Skills |
| **OAuth 2.1 + PKCE** | 所有连接器的身份验证 |
| **OpenTelemetry** | 标准化可观测性 |
| **AGPL-3.0** | 完整源代码，可审计，可修改 |

### 9.3. 可扩展性

每位用户都可以连接自己的MCP服务器，将LIA的能力扩展到内置工具之外。Skills（agentskills.io标准）允许以自然语言注入专家指令——内置的Skills生成器让创建新技能变得简单。从 v1.16.8 开始，一个 Skill 还可以在聊天中直接返回**交互式 HTML 框架**（地图、仪表板、日历、换算器……）或**图像**（QR 码、图表），在严格的 CSP 沙盒中运行，主题和语言自动同步。

LIA的架构专为方便扩展而设计——添加新的连接器、频道、智能体和AI提供商都有清晰的抽象层和专属开发指南（智能体创建指南、工具创建指南），让任何开发者都能轻松上手扩展。

### 9.4. 多渠道

响应式网页界面之外，还有原生Telegram集成（对话、语音消息转录、内联审批按钮、主动通知）以及Firebase推送通知。你的记忆、日志和偏好设置会跨渠道同步。

## 10. 愿景

### 10.1. 与你一同成长的智能

心理记忆、内省日志、贝叶斯学习与Psyche Engine的组合，创造出一种涌现式智能：随着时间推移，LIA对你是谁的理解会越来越细腻深刻。这不是通用人工智能——而是一种**实用的、关系性的、情感性的智能**，服务于一个具体的人。

### 10.2. LIA 不是什么

LIA不是云端巨头的竞争者，也不打算与其研究预算相抗衡。作为纯粹的对话聊天机器人，这些模型通过官方界面使用时可能会更流畅。但LIA不是聊天机器人——它是一个**智能编排系统**，将这些模型作为组件加以调用，一切尽在你的掌控之中。

### 10.3. LIA 存在的理由

LIA的存在，是因为这个世界缺少一个真正**属于你**的AI助手。真正属于你。日常管理简单。可以与你的家人朋友共享，每个人都有独特的情感关系。托管在你自己的服务器上。对每一个决策和每一分成本都透明公开。能够提供商业助手所没有的情感深度。在生产环境中稳定可靠。并且完全开放——对提供商、对标准、对代码。
